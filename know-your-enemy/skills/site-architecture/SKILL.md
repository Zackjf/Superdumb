# Site Architecture Analysis

Count what pages exist across all visited sites. The numbers tell you what to build.

## What to Count

For each page type, count how many of your visited sites have it:

| Page type | Present on X of Y sites | Verdict |
|---|---|---|
| Homepage | Y/Y | Table stakes (always) |
| Individual service pages | ?/Y | If 10+: table stakes. If 5-9: recommended. |
| Single services list | ?/Y | Note: do the individual-page sites rank better? |
| About / Team | ?/Y | Almost always table stakes for service businesses |
| Contact | ?/Y | Table stakes (always) |
| Blog | ?/Y | Count ACTIVE blogs (posted in last 90 days) separately |
| FAQ | ?/Y | Check if FAQ sites capture more long-tail keywords |
| Testimonials (separate page) | ?/Y | vs embedded on homepage only |
| Gallery / portfolio | ?/Y | Industry-dependent |
| Pricing page | ?/Y | Do sites that show prices convert better? |
| Case studies | ?/Y | Common for B2B and professional services |
| Privacy policy | ?/Y | Cross-reference with legal requirements |
| Terms of service | ?/Y | Cross-reference with legal requirements |
| Location pages | ?/Y | Important for multi-location businesses |
| Careers page | ?/Y | Only for companies actively hiring |

## Individual vs Aggregated Pages

A critical question for service businesses: do competitors have ONE services page or INDIVIDUAL pages per service?

Check which approach correlates with higher rankings. If "teeth whitening [city]" is won by sites with a dedicated `/whitening` page rather than sites with a single `/services` page, the recommendation is individual pages.

## Output

A ranked page list:
- **Required** (12+ of 15): build these
- **Recommended** (7-11 of 15): strong case to build
- **Optional** (3-6 of 15): build if it differentiates
- **Not needed** (<3 of 15): explicitly skip, state why
