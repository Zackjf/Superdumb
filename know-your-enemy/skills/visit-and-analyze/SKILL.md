# Visit & Analyze

Visit every site on the research list. For each site, take structured notes using the template below. This is the raw data that the synthesis phase turns into recommendations.

## How to Visit

Use WebFetch to visit each site. For each site, visit at minimum:
1. **Homepage** — the most important page. Note everything.
2. **Services/products page** — how they present what they offer
3. **About page** — how they build trust
4. **Contact page** — how they convert visitors
5. **Blog** (if it exists) — check if it's active, what they write about
6. **Any page that looks unique** — before/after gallery, FAQ, pricing, testimonials

Don't try to visit every page on every site. Visit the 4-6 most important pages, take notes, move on. Depth on 5 pages beats shallow notes on 20.

## Per-Site Analysis Template

For each site, record:

```
SITE: [URL]
TYPE: [Direct competitor / Best-in-class / Adjacent]
BUSINESS: [What they do, where]
FIRST IMPRESSION: [Professional / Modern / Outdated / Sparse / Overwhelming]

PAGES FOUND:
- [x] Homepage
- [x] Services (individual pages / single list / grid)
- [x] About / Team
- [x] Contact
- [ ] Blog (active / abandoned / none)
- [ ] FAQ
- [ ] Testimonials / Reviews
- [ ] Gallery / Portfolio
- [ ] Pricing
- [ ] Legal (privacy / terms)
- [ ] Other: ___

NAVIGATION:
- Items: [list the nav items, exact text]
- Count: [number]
- CTA in nav? [yes/no, what text]
- Dropdown/mega-menu? [yes/no]
- Mobile: [hamburger / bottom nav / sticky CTA / not checked]

CONVERSION:
- Primary CTA: [text, e.g., "Agendar cita"]
- Contact channels: [WhatsApp / Phone / Form / Email / Booking / Chat]
- Floating element: [WhatsApp button / chat widget / none]
- Form fields: [list the fields on the contact/booking form]
- Clicks from homepage to contact: [count]

TRUST SIGNALS:
- Credentials shown? [degrees, certifications, badges]
- Reviews/testimonials? [Google widget / text quotes / video / none]
- Real photos? [team / facility / before-after / stock only]
- Social proof? ["15 years experience" / "5000 patients" / etc.]
- Professional associations? [logos, badges]

CONTENT:
- Blog: [active (last post date) / abandoned / none]
- Blog topics: [what they write about]
- FAQ: [yes/no, how many questions]
- Case studies: [yes/no]
- Video: [embedded / linked / none]
- Resources/guides: [yes/no]

LEGAL PAGES:
- Privacy policy: [yes/no]
- Terms of service: [yes/no]
- Cookie consent: [banner / none]
- Industry disclaimers: [what, if any]
- Accessibility statement: [yes/no]

PRICING:
- Prices shown? [yes with numbers / "from $X" / "contact us" / none]

VISUAL:
- Hero: [image / video / slider / text-only]
- Photography: [real / stock / mixed]
- Design quality: [1-5]
- Page speed feel: [fast / medium / slow]

- [Anything unique or interesting about this site]
- [Anything notably bad]
- [Anything you'd steal for the client's site]
```

## Efficiency Rules

- **Don't spend more than 3-5 minutes per site.** You're pattern-matching, not doing a full audit.
- **Skip sites that are clearly irrelevant** (a competitor's site is just a Facebook page with no real website → note "social-only, no website" and move on).
- **Note what's MISSING as much as what's present.** "No FAQ, no blog, no prices shown" is valuable data.
- **Visit the best-in-class sites more carefully** — these are the aspirational benchmark.
- **For adjacent businesses, focus on LOCAL patterns** — contact channels, price display, design quality baseline. Don't analyze their content deeply.

## Parallel Visits

If possible, visit multiple sites in parallel using subagents. Give each subagent 3-5 sites and the analysis template. Merge the results.

## Output

A collection of 15-25 site analyses using the template above. These feed into the synthesis phase where patterns are extracted across all sites.
